# reports

P&L calculation, report generation, and report rendering for end-of-day, weekly, and monthly summaries.

## Structure

```
reports/
├── pnl.py      # FIFO P&L computation
├── fetch.py    # Historical data fetching for report generation
├── engine.py   # Report engine — orchestrates fetch + compute + render
└── render.py   # Report rendering (console / Telegram message)
```

## P&L model (`pnl.py`)

Implements FIFO (first-in, first-out) trade matching. Sells are matched against the oldest open buy lots. This gives the correct cost basis for trades where position is built up over multiple partial fills.

```python
def compute_pnl(trades: list[Trade]) -> PnlResult
```

Returns total realized PnL, per-trade breakdown, and average holding time.

> Note: `Repository.get_daily_realized_pnl()` uses a simpler cash-flow sum (sum of `price × qty` by side) which is fast enough for the intraday risk check. The FIFO method here is used only for reports where accuracy matters.

## Report types

Reports are generated by the `scripts/` CLI tools:

| Script | Report |
|--------|--------|
| `day_end_report.py` | Today's trades, PnL, open positions |
| `week_report.py` | Weekly summary with per-strategy breakdown |
| `month_report.py` | Monthly summary |

All three share the same engine; the time range is the only difference.

## Output

Reports render to:
- Console (always)
- Telegram (if `TELEGRAM_BOT_TOKEN` is configured)
